# Client Summary Writing Guide

Use this format for client-facing technical summaries.

Template:

```
Title (no formatting)

Problem:
[Clear problem statement]

Solution:
[Simple description of what was built - no buzzwords]

[Specific Results with numbers/endpoints]

Technical Evidence:
[Concrete proof - actual error codes, response messages]

[Clear next action for recipient]
```

Requirements:

- No markdown formatting, bold text, or visual noise.
- No corporate buzzwords (for example: "comprehensive", "professional", "enterprise-grade").
- Use simple, direct language describing exactly what was done.
- Include concrete technical evidence.
- End with a clear call to action.
- Keep to one page maximum.
- Keep claims factual and non-inflated.
